## 3. Key Trading Constraints (CRITICAL)

Each trading pair includes:

### PricePrecision

- Number of decimal places allowed for price
- Example: `2 → 42000.12`
- Must round before sending orders

### AmountPrecision

- Number of decimal places allowed for quantity
- Example: `5 → 0.00001 BTC`

### MiniOrder

- Minimum notional size:

    ```
    price × quantity ≥ 1 USD
    ```

### CanTrade

- If `false`, trading is disabled for that pair
